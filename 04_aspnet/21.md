<div align="center"><img src="https://i.imgur.com/pJkUW9q.png" title="source: imgur.com" /></div>

**Linha 01:** Importamos o Namespace **blogpessoal** (pasta raﾃｭz do projeto Blog Pessoal), atravﾃｩs da palavra reservada **using**.

**Linha 02:** Importamos o Namespace **Data** (pasta Model do projeto Blog Pessoal), atravﾃｩs da palavra reservada **using**.

**Linha 03:** Importamos o Namespace **AspNetCore.Hosting**, atravﾃｩs da palavra reservada **using**. Este Pacote possui os Mﾃｩtodos necessﾃ｡rios para configurar serviﾃｧos adicionais para o aplicativo Web.

**Linha 04:** Importamos o Namespace **AspNetCore.Mvc.Testing**, atravﾃｩs da palavra reservada **using**. Este Pacote possui os Mﾃｩtodos necessﾃ｡rios para implementar testes em aplicaﾃｧﾃｵes MVC.

**Linha 05:** Importamos o Namespace **Microsoft.AspNetCore.TestHost**, atravﾃｩs da palavra reservada **using**. Este Pacote possui os Mﾃｩtodos necessﾃ｡rios para implementar o Servidor de testes.

**Linha 06:** Importamos o Namespace **EntityFrameworkCore**, atravﾃｩs da palavra reservada **using**. Este Pacote possui os Mﾃｩtodos necessﾃ｡rios para interagir com o Banco de dados.

**Linha 07:** Importamos o Namespace **Extensions.DependencyInjection**, atravﾃｩs da palavra reservada **using**. Este Pacote implementa Serviﾃｧos como uma Injeﾃｧﾃ｣o de Dependﾃｪncias.

**Linha 08:** Importamos o Namespace **WebMotions.Fake.Authentication.JwtBearer**, atravﾃｩs da palavra reservada **using**. Este Pacote implementa o Serviﾃｧo de geraﾃｧﾃ｣o de Token JWT para testar endpoints protegidos.

**Linha 12:** Foi criada a Classe **WebAppFactory** como uma Heranﾃｧa da Classe **WebApplicationFactory**. `WebApplicationFactory<TEntryPoint>` ﾃｩ usado para criar um Servidor de Testes, para executar os testes de integraﾃｧﾃ｣o e End to End. O Generic **TEntryPoint** ﾃｩ a classe de ponto de entrada da aplicaﾃｧﾃ｣o que serﾃ｡ testada, ou seja, a Classe **Program**.

As classes de teste implementarﾃ｣o uma interface auxiliar, chamada **IClassFixture**, que indica que a classe contﾃｩm testes e fornece instﾃ｢ncias de objetos compartilhadas entre os testes na classe, ou seja, contﾃｩm dependﾃｪncias que nﾃｳs utilizaremos nos testes. Em nosso projeto, iremos compartilhar o Banco de dados em memﾃｳria e o Serviﾃｧo de Geraﾃｧﾃ｣o de Tokens JWT (Fake Token).

A classe de testes, usa a Classe **WebAppFactory** para iniciar o projeto de testes do xUnit e fornecer um Cliente HTTP, atravﾃｩs da Classe **HttpClient**, para os mﾃｩtodos de teste realizarem Requisiﾃｧﾃｵes HTTP.

**Linha 14:** Cria o Mﾃｩtodo **ConfigureWebHost(IWebHostBuilder builder)**, que cria uma nova configuraﾃｧﾃ｣o para a aplicaﾃｧﾃ｣o, atravﾃｩs da sobrecarga do Mﾃｩtodo.

**Linha 17:** Cria um serviﾃｧo adicional, que poderﾃ｡ ser chamado inﾃｺmeras vezes. Aqui vamos construir a configuraﾃｧﾃ｣o do Banco de dados em memﾃｳria, que serﾃ｡ utilizado nos testes.

**Linha 19:** Realiza uma busca nos serviﾃｧos da aplicaﾃｧﾃ｣o, para localizar a configuraﾃｧﾃ｣o do contexto padrﾃ｣o aplicaﾃｧﾃ｣o, ou seja, o Banco de dados.

**Linhas 23 e 24:** Caso o contexto da aplicaﾃｧﾃ｣o seja encontrado, ele serﾃ｡ removido.

**Linhas 26 a 29:** Adiciona um novo contexto contendo a configuraﾃｧﾃ｣o do Banco de dados em memﾃｳria. Essa configuraﾃｧﾃ｣o substituirﾃ｡ o contexto padrﾃ｣o da aplicaﾃｧﾃ｣o durante a execuﾃｧﾃ｣o do teste.

**Linhas 31 a 33:** Registra o novo contexto no projeto de testes.

**Linhas 35 a 43:** Tenta criar as Tabelas no Banco de dados em Memﾃｳria, baseados na Classe **AppDbContext**, que define o contexto da aplicaﾃｧﾃ｣o principal. Caso as tabelas nﾃ｣o sejam criadas, serﾃ｡ retornada uma mensagem de erro.

**Linha 47:** Especifica a pasta raiz do projeto web como a pasta principal do aplicativo, quando o aplicativo for iniciado a partir desta pasta. O ponto indica que a pasta principal ﾃｩ a pasta do projeto.

**Linha 49:** Cria um Servidor de Testes e substitui o serviﾃｧo de geraﾃｧﾃ｣o do Token JWT, que foram adicionados na classe Program, pelo Serviﾃｧo de Geraﾃｧﾃ｣o de Token JWT Fake Token.

**Linha 51:** Adicionamos o serviﾃｧo de  autenticaﾃｧﾃ｣o ao pipeline de execuﾃｧﾃ｣o do Projeto de testes do ASP.NET, ou seja, habilitamos o  serviﾃｧo de autenticaﾃｧﾃ｣o Fake Token, que irﾃ｡ procurar no Cabeﾃｧalho (header) de todas as requisiﾃｧﾃｵes HTTP de teste por um Token JWT.

**Linha 53:** Estamos informando ao serviﾃｧo de autenticaﾃｧﾃ｣o, que ele deve procurar um Token **Bearer**, no formato **JWT**. **Bearer** significa que o Token serﾃ｡ enviado no cabeﾃｧalho da requisiﾃｧﾃ｣o, na propriedade **Authorization**.

**Linha 54:** O Token JWT pode ser gerado de vﾃ｡rias formas, inclusive em sites externos, como: Facebook, Twitter, Google e  Microsoft. Este processo ﾃｩ chamado de **Challenge** ou desafiar, onde confrontamos um servidor para que ele faﾃｧa uma autenticaﾃｧﾃ｣o e nos retorne o Token. Desta forma, nosso desafio  aqui ﾃｩ dizer para o ASP.NET que estamos utilizando uma autenticaﾃｧﾃ｣o  interna, ou seja, ela ﾃｩ gerada neste servidor e vale ﾃｺnica e  exclusivamente para este servidor. Por este motivo, precisamos de uma  configuraﾃｧﾃ｣o adicional **DefaultChallengeScheme**, que  definirﾃ｡ este processo.

**Linhas 55:** Inicializa o Serviﾃｧo Fake Token com as opﾃｧﾃｵes padrﾃ｣o.

**Linha 57:** Inicializa o Projeto de testes com todas as configuraﾃｧﾃｵes criadas anteriormente, ou seja, com o Banco de dados em memﾃｳria e com o serviﾃｧo Fake Token.

O Cﾃｳdigo completo, vocﾃｪ confere abaixo:

```c#
using blogpessoal;
using blogpessoal.Data;
using Microsoft.AspNetCore.Hosting;
using Microsoft.AspNetCore.Mvc.Testing;
using Microsoft.AspNetCore.TestHost;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;
using WebMotions.Fake.Authentication.JwtBearer;

namespace blogpessoaltest.Factory
{
    public class WebAppFactory : WebApplicationFactory<Program>
    {
        protected override void ConfigureWebHost(IWebHostBuilder builder)
        {

            builder.ConfigureServices(services =>
            {
                var descriptor = services.SingleOrDefault(
                    d => d.ServiceType ==
                        typeof(DbContextOptions<AppDbContext>));

                if (descriptor != null)
                    services.Remove(descriptor);

                services.AddDbContext<AppDbContext>(options =>
                {
                    options.UseInMemoryDatabase("InMemoryBlogPessoalTest");
                });

                var sp = services.BuildServiceProvider();
                using var scope = sp.CreateScope();
                using var appContext = scope.ServiceProvider.GetRequiredService<AppDbContext>();

                try
                {
                    appContext.Database.EnsureCreated();
                }
                catch (Exception ex)
                {
                    Console.WriteLine(ex.ToString());
                    throw;
                }

            });

            builder.UseContentRoot(".");

            builder.UseTestServer().ConfigureTestServices(collection =>
            {
                collection.AddAuthentication(options =>
                {
                    options.DefaultAuthenticateScheme = FakeJwtBearerDefaults.AuthenticationScheme;
                    options.DefaultChallengeScheme = FakeJwtBearerDefaults.AuthenticationScheme;
                }).AddFakeJwtBearer();
            });
            base.ConfigureWebHost(builder);
        }
    }
}
```
